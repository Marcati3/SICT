All output files for Business-SICT must be saved into the appropriate **department subfolder**, not directly into `Projects/Business-SICT/`.

Existing department folders: HR, Marketing, Sales, Competitive-Intel, Opportunities, SIC-Dashboards, IT

**Rule:** Match the output to the responsible team:
- IT work (SEO fixes, schema, server config, technical) → `IT/`
- Marketing deliverables → `Marketing/`
- HR documents → `HR/`
- Sales materials → `Sales/`

Within each department folder, create a descriptive subfolder for the workstream (e.g., `IT/SEO-Audit/`).

This was explicitly corrected by the user on 2026-03-16 after files were wrongly saved to the project root.
